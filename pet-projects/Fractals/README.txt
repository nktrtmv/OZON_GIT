1 В ридми расскажу о некоторых моментах работы программы и об ограничениях, установленных мною.
2 Для каждого фрактала свои настройки длины отрезка и глубины рекурсии.
3 При изменении глубины рекурсии фрактал перерисовывается только если уже нарисован.
4 После выбора другого фрактала его нужно снова нарисовать по кнопке.
5 Некорректные действия - это сохранение фрактала если он не нарисован или построение фрактала если не выбран его тип.
6 По поводу пункта 5 (доп функционал):
	Никаких условий как должен перерисовываться фрактал при изменении окна не добавлено, поэтому при изменении
	размера окна у меня фрактал перерисовывается так, чтобы он был всегда в центре изображения и не съезжал
	при изменении размеров. Окно масштабируемо и вся компоновка на гриде, чтобы масштабирование было.
	По поводу минимального и максимального размерах написано только "можно", поэтому сделал как сам пожелал.
7 Цветовой градиент сделан для всех фракталов и цвета генерируются по закону интерполяции цвета.
8 Для изменения масштаба можно использовать слайдер сверху приложения (0.75х - 7.5х). (2,3,5 включаются в этот промежуток)
9 Для перемещеня изображения сделан scrollviewer, причем при изменении масштаба видно центральную часть для удобства.
10 Фрактал сохраняется в формате png только если он построен, сохраняется в папке Save в проекте в .net5-windows.

Несколько слов о работе программы:
1 выбор цвета градиента реализован с помощью ползунков отвечающих за цвета RGB.
2 Для выбора фрактала нужно нажать на кнопочку сверху (их 5).
3 Для каждого фрактала выбрана такая максимальная глубина рекурсии, чтобы ничего не лагало и не падало.
	Специально проверял, чтобы не было переполнения стэка и прочего на слабом компьютере и падать не будет не при
	каких данных.
4 Чтобы поиграться (не беру на себя ответственность, это вне рамках пиргрейда и оценки) можно изменить свойства в каждом из
	классов и установить большую глубину рекурсии, например я строил дерево с глубиной 28.)
5 Ограничения по углам в дереве -179:179, по отношению длин отрезков двух итераций установил ограничение 0.25-0.75, чтобы
	не ломать интерфейс выходом за границы канваса, и чтобы фрактал был видел, так как при маленьком отношении
	ничего не видно из за маленького масштаба, а при большом отношении можно разглядеть фрактал только на маленькой
	длине первой линии.
6 Первые итерации для дерева и кривой - это линии, для ковра Серый квадрат(как полотно) и первая итерация, то есть
	первый квадрат с начальным цветом градиента в центре, для треугольника - это треугольник (ВАУ), и для
	множества Кантора это первый сегмент-прямоугольник(-линия?)

Минимальные требования к компъютеру: 8ГБ оперативной памяти, 4-ядерный процессор 2016 года и новее, SSD
